# RW_1.1: TaskLifecycleStatus — Implementation Spec

## Approval

- [ ] Approved

## Purpose

Fuehrt `lifecycleStatus` auf `LocalTask` ein, damit Quick-Capture-Tasks (`.raw`) vom Backlog (`.active`) getrennt werden. Grundlage fuer Story 1.3 (Refiner).

## Design-Entscheidungen

### D1: String-Storage (wie `urgency`/`taskType`)

```swift
// Enum (Semantic Wrapper — nicht auf LocalTask gespeichert)
enum TaskLifecycleStatus: String, Codable, CaseIterable {
    case raw       // Gerade erfasst, nicht veredelt
    case refined   // Durch Refiner bestaetigt
    case active    // Im Backlog / Next-Up
}

// LocalTask Property (String fuer CloudKit-Kompatibilitaet)
var lifecycleStatus: String = "active"
```

**Begruendung:** Folgt dem etablierten Pattern (`taskType: String` + `TaskCategory` Enum). CloudKit braucht primitive Typen mit Default.

### D2: Default = `"active"` (nicht `"raw"`)

Bestehende Tasks (SwiftData Lightweight Migration) bekommen automatisch `"active"` — sie bleiben im Backlog sichtbar. Kein VersionedSchema noetig.

### D3: Filter in `fetchIncompleteTasks()`

```swift
// LocalTaskSource.fetchIncompleteTasks()
return allIncomplete.filter { $0.isVisibleInBacklog && $0.lifecycleStatus != "raw" }
```

Zentraler Filter — BacklogView, SyncEngine, macOS ContentView profitieren automatisch.

### D4: Quick Capture → `.raw`, TaskFormSheet → `.active`

| Entry Point | lifecycleStatus | Begruendung |
|-------------|----------------|-------------|
| QuickCaptureView (iOS) | `.raw` | Schnellerfassung ohne Formular |
| QuickCapturePanel (macOS) | `.raw` | Spotlight-Style, nur Textfeld |
| CreateTaskIntent (Siri) | `.raw` | Freitext, keine Metadaten |
| ShareViewController (iOS) | `.raw` | Externer Input |
| MacShareSheetView (macOS) | `.raw` | Externer Input |
| VoiceInputSheet (watchOS) | `.raw` | Spracheingabe |
| TaskFormSheet (iOS/macOS) | `.active` | Bewusste Erstellung mit Formular |
| RecurrenceService | `.active` | Wiederkehrende Instanzen sind bereits veredelt |

### D5: Uebergangs-Fallback (bis Refiner existiert)

Bearbeitung eines `.raw`-Tasks via TaskFormSheet setzt `lifecycleStatus = "active"`.
Damit kann der User `.raw`-Tasks manuell ins Backlog befoerdern, bis Story 1.3 implementiert ist.

## Implementation Details

### 1. LocalTask.swift

```swift
// Nach den bestehenden Enums (vor der @Model class)
enum TaskLifecycleStatus: String, Codable, CaseIterable {
    case raw
    case refined
    case active
}

// In der @Model class, nach sourceURL
/// Lifecycle status: raw (just captured) → refined (reviewed) → active (in backlog)
var lifecycleStatus: String = "active"

// Init: neuer Parameter mit Default
init(
    ...
    lifecycleStatus: String = "active"
) {
    ...
    self.lifecycleStatus = lifecycleStatus
}
```

### 2. PlanItem.swift

```swift
// Property
let lifecycleStatus: String

// In init(localTask:)
self.lifecycleStatus = localTask.lifecycleStatus

// In init(reminder:metadata:) — Reminders sind immer active
self.lifecycleStatus = "active"
```

### 3. LocalTaskSource.swift

```swift
// createTask() — neuer Parameter
func createTask(
    ...
    lifecycleStatus: String = "active"
) async throws -> LocalTask {
    ...
    let task = LocalTask(
        ...
        // lifecycleStatus wird nach Erstellung gesetzt
    )
    task.lifecycleStatus = lifecycleStatus
    ...
}

// fetchIncompleteTasks() — Filter erweitern
return allIncomplete.filter { $0.isVisibleInBacklog && $0.lifecycleStatus != "raw" }
```

### 4. Quick Capture Entry Points (je 1 Zeile)

```swift
// QuickCaptureView.saveTask() — nach task-Erstellung
task.lifecycleStatus = TaskLifecycleStatus.raw.rawValue

// QuickCapturePanel.addTask() — nach task-Erstellung
task?.lifecycleStatus = TaskLifecycleStatus.raw.rawValue

// CreateTaskIntent.perform() — nach LocalTask(title:)
task.lifecycleStatus = TaskLifecycleStatus.raw.rawValue

// ShareViewController (iOS + macOS) — nach LocalTask(title:)
task.lifecycleStatus = TaskLifecycleStatus.raw.rawValue

// VoiceInputSheet (watchOS) — nach LocalTask(title:)
task.lifecycleStatus = TaskLifecycleStatus.raw.rawValue
```

## Known Limitations

- **Kein Refiner:** Bis Story 1.3 gibt es keinen dedizierten UI-Flow fuer `.raw` → `.active`. Fallback: TaskFormSheet-Bearbeitung
- **Kein Badge:** Story 1.1 Spec erwaehnt Badge am Refiner-Tab — wird mit Story 1.3 implementiert
- **Share Extensions:** Haben eigene ModelContainer-Instanzen, setzen `lifecycleStatus` direkt (kein LocalTaskSource)
