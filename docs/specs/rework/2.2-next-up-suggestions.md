# Story 2.2: KI-gestützte Tagesvorschläge

**Epic:** 2 — Adaptive Tagesplanung
**Priorität:** Mittel | **Abhängigkeiten:** Story 0.2, Story 2.1

## Ziel

Morgens 3-5 Task-Vorschläge basierend auf freien Kalender-Slots, Verhaltens-Profil und Task-Priorität.

## Kontext

Die Intelligenz kommt NICHT aus dem LLM, sondern aus dem `BehavioralProfileService` (deterministisch). Das LLM wird nur optional für die Formulierung eingesetzt.

## Vorschlags-Logik (deterministisch)

Für jede Kalender-Lücke:
1. Filtere Tasks die in die Lücke passen (`estimatedDuration <= Lücken-Dauer`)
2. Filtere Tasks ohne unerfüllte Blocker (`blockerIDs`)
3. Score pro Task = `priorityScore` × `timeAffinityBonus` × `rescheduleBonus`
   - `timeAffinityBonus`: Passt die Task-Kategorie zur Tageszeit? (aus BehavioralProfile)
   - `rescheduleBonus`: Oft verschobene Tasks werden höher gewichtet
4. Kalender-Korrelation: An Tagen mit vielen Meetings weniger/kürzere Tasks vorschlagen
5. Top-Task pro Lücke wird Vorschlag

## Akzeptanzkriterien

- [ ] Morgens (oder bei erstem App-Start des Tages) werden 3-5 Vorschläge generiert
- [ ] Vorschläge sind an konkrete Kalender-Lücken gebunden (Zeitfenster sichtbar)
- [ ] Nutzer kann pro Vorschlag: Tap bestätigen → Task wird als Next-Up + scheduled markiert
- [ ] Nutzer kann pro Vorschlag: Swipe → Vorschlag verschwindet
- [ ] Nutzer kann eigene Tasks manuell zu Next Up hinzufügen (unverändert zum bisherigen Flow)
- [ ] Vorschläge sind optional, vollständig ignorierbar
- [ ] Ohne Apple Intelligence: Vorschläge basieren nur auf deterministischem Scoring
- [ ] Gesamte geschätzte Dauer der Vorschläge ≤ verfügbare freie Zeit
- [ ] Vorschläge werden einmal pro Tag berechnet und gecacht

## Technische Vorgaben

### Neue Dateien
| Datei | Beschreibung |
|---|---|
| `NextUpSuggestionService.swift` | Orchestriert: BehavioralProfile + GapFinder + TaskPriorityScoringService |

### Betroffene Dateien
| Datei | Änderung |
|---|---|
| `MorningCoachingSection.swift` | Zeigt Vorschläge (konsumiert NextUpSuggestionService) |
| `GapFinder.swift` | Keine Änderung, wird konsumiert |
| `AITaskScoringService.swift` | Optional: kann Ranking verfeinern, ist aber nicht Kern-Logik |

### Constraints
- Vorschläge dürfen keine Tasks mit unerfüllten Blockern enthalten
- Berechnung muss < 200ms dauern
- Cache in-memory, Invalidierung bei Task-Änderungen
