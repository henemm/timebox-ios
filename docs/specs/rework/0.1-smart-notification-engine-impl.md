# RW_0.1 Phase A — Smart Notification Engine Foundation

## Approval

- [ ] Approved

## Purpose

Baut den `SmartNotificationEngine` als orchestrierenden Wrapper-Layer über dem bestehenden `NotificationService`. Phase A legt das Foundation-Fundament: Engine Core, Profil-Setting in `AppSettings`, und die Anbindung in `FocusBloxApp` als ersten Reconciliation-Trigger — ohne bestehende Views oder Services anzufassen.

## Source

- **File:** `Sources/Services/SmartNotificationEngine.swift` (CREATE)
- **Identifier:** `enum SmartNotificationEngine`
- **Weitere Files:** `Sources/Models/AppSettings.swift` (MODIFY), `Sources/FocusBloxApp.swift` (MODIFY)

## Dependencies

| Entity | Type | Purpose |
|--------|------|---------|
| `NotificationService` | service | Liefert `build*Request`-Methoden — Engine nutzt sie intern, löst sie nicht ab |
| `AppSettings` | model | Liest `notificationProfile` für Budget-Entscheidungen |
| `UNUserNotificationCenter` | system | `removeAllPendingNotificationRequests()` + `add(_:)` pro Reconciliation |
| `ModelContainer` / `ModelContext` | swiftdata | Fetch offener Tasks mit `dueDate` für Prio-2-Slots |
| `EventKitRepository` | service | `fetchFocusBlocks(for:)` für Prio-1-Timer-Notifications |
| `LocalTask` | model | Datenquelle: `dueDate`, `scheduledDate`, `isCompleted`, `isNextUp` |
| `FocusBlock` | model | Datenquelle: `startDate`, `endDate`, `id` für Timer-Notifications |

## Implementation Details

### 1. `Sources/Services/SmartNotificationEngine.swift` (~150 LoC, CREATE)

```swift
import UserNotifications
import SwiftData
#if canImport(BackgroundTasks) && !os(macOS)
import BackgroundTasks
#endif

/// Zentraler Notification-Orchestrator. Berechnet bei jedem Trigger
/// die gesamte Notification-Queue neu (Reconcile-on-Event-Strategie).
/// Nutzt intern NotificationService.build*Request-Methoden.
@MainActor
enum SmartNotificationEngine {

    // MARK: - Enums

    enum NotificationProfile: String, CaseIterable {
        case quiet       = "quiet"        // Leise: nur Focus-Timer
        case balanced    = "balanced"     // Ausgeglichen: Timer + Tasks + Review
        case active      = "active"       // Aktiv: alles inkl. Nudges
    }

    enum ReconciliationReason {
        case appForeground
        case appBackground
        case taskChanged
        case blockChanged
        case profileChanged
    }

    // MARK: - Budget Constants (Prio 1-5, total = 64)

    private static let budgetTimers: Int = 4       // Prio 1
    private static let budgetTasks: Int  = 20      // Prio 2
    private static let budgetReview: Int = 2       // Prio 3
    private static let budgetNudges: Int = 10      // Prio 4
    // Reserve = 28 (Puffer, ungeplant)

    // MARK: - BGAppRefreshTask

    static let bgTaskIdentifier = "com.henning.focusblox.notification-refresh"

    // MARK: - Reconcile Entry Point

    /// Berechnet alle Notifications neu. Muss auf einem non-blocking Task aufgerufen werden.
    /// - Parameters:
    ///   - reason: Warum die Reconciliation ausgelöst wurde (für Logging)
    ///   - container: SwiftData ModelContainer für Task-Fetch
    ///   - eventKitRepo: EventKitRepository für FocusBlock-Fetch
    static func reconcile(
        reason: ReconciliationReason,
        container: ModelContainer,
        eventKitRepo: EventKitRepository
    ) async {
        let start = Date()
        let center = UNUserNotificationCenter.current()
        let profile = AppSettings.shared.notificationProfile

        // Schritt 1: Alle pending Notifications löschen
        center.removeAllPendingNotificationRequests()

        var requests: [UNNotificationRequest] = []

        // Schritt 2: Prio 1 — Focus Block Timer (alle Profile)
        requests += buildTimerRequests(eventKitRepo: eventKitRepo)

        // Schritt 3: Prio 2 — Task-Erinnerungen (Ausgeglichen + Aktiv)
        if profile == .balanced || profile == .active {
            requests += buildTaskRequests(container: container)
        }

        // Schritt 4: Prio 3 — Abend-Review + Morning (Ausgeglichen + Aktiv)
        if profile == .balanced || profile == .active {
            requests += buildReviewRequests()
        }

        // Schritt 5: Prio 4 — Nudges (nur Aktiv)
        if profile == .active {
            requests += buildNudgeRequests()
        }

        // Schritt 6: Requests einplanen (max 64 gesamt)
        let capped = Array(requests.prefix(64))
        for request in capped {
            try? await center.add(request)
        }

        let elapsed = Date().timeIntervalSince(start) * 1000
        print("SmartNotificationEngine: reconcile(\(reason)) — \(capped.count) notifications, \(String(format: "%.1f", elapsed))ms")
    }

    // MARK: - Prio 1: Timer Requests

    private static func buildTimerRequests(
        eventKitRepo: EventKitRepository
    ) -> [UNNotificationRequest] {
        let today = Date()
        let tomorrow = Calendar.current.date(byAdding: .day, value: 1, to: today) ?? today
        var blocks: [FocusBlock] = []
        blocks += (try? eventKitRepo.fetchFocusBlocks(for: today)) ?? []
        blocks += (try? eventKitRepo.fetchFocusBlocks(for: tomorrow)) ?? []

        let now = Date()
        var requests: [UNNotificationRequest] = []

        for block in blocks.prefix(2) { // max aktiver + nächster Block = 4 Slots
            if let startReq = NotificationService.buildFocusBlockNotificationRequest(
                blockID: block.id,
                blockTitle: block.title,
                startDate: block.startDate,
                minutesBefore: 5,
                now: now
            ) {
                requests.append(startReq)
            }
            if let endReq = NotificationService.buildFocusBlockEndNotificationRequest(
                blockID: block.id,
                blockTitle: block.title,
                endDate: block.endDate,
                completedCount: block.completedTaskIDs.count,
                totalCount: block.taskIDs.count,
                now: now
            ) {
                requests.append(endReq)
            }
            if requests.count >= budgetTimers { break }
        }

        return Array(requests.prefix(budgetTimers))
    }

    // MARK: - Prio 2: Task Requests

    private static func buildTaskRequests(
        container: ModelContainer
    ) -> [UNNotificationRequest] {
        let context = ModelContext(container)
        let settings = AppSettings.shared
        guard settings.dueDateMorningReminderEnabled || settings.dueDateAdvanceReminderEnabled else {
            return []
        }

        let descriptor = FetchDescriptor<LocalTask>(
            predicate: #Predicate<LocalTask> { !$0.isCompleted && !$0.isTemplate && $0.dueDate != nil },
            sortBy: [SortDescriptor(\.dueDate)]
        )
        guard let tasks = try? context.fetch(descriptor) else { return [] }

        let now = Date()
        var requests: [UNNotificationRequest] = []

        for task in tasks {
            guard let dueDate = task.dueDate, dueDate > now else { continue }
            if settings.dueDateMorningReminderEnabled,
               let req = NotificationService.buildDueDateMorningRequest(
                   taskID: task.id,
                   title: task.title,
                   dueDate: dueDate,
                   morningHour: settings.dueDateMorningReminderHour,
                   morningMinute: settings.dueDateMorningReminderMinute,
                   now: now
               ) {
                requests.append(req)
            }
            if settings.dueDateAdvanceReminderEnabled,
               let req = NotificationService.buildDueDateAdvanceRequest(
                   taskID: task.id,
                   title: task.title,
                   dueDate: dueDate,
                   advanceMinutes: settings.dueDateAdvanceReminderMinutes,
                   now: now
               ) {
                requests.append(req)
            }
            if requests.count >= budgetTasks { break }
        }

        return Array(requests.prefix(budgetTasks))
    }

    // MARK: - Prio 3: Review / Morning Requests

    private static func buildReviewRequests() -> [UNNotificationRequest] {
        // Phase A: Platzhalter — Review/Morning-Notifications werden in Phase C implementiert.
        // Identifier-Schema: focusblox.review.{date}, focusblox.morning.{date}
        return []
    }

    // MARK: - Prio 4: Nudge Requests

    private static func buildNudgeRequests() -> [UNNotificationRequest] {
        // Phase A: Platzhalter — Nudge-Notifications werden in Phase C implementiert.
        // Identifier-Schema: focusblox.nudge.{type}.{date}
        return []
    }

    // MARK: - BGAppRefreshTask Registration (iOS only)

    #if !os(macOS)
    /// Registriert einen BGAppRefreshTask als Best-Effort-Aufräumer.
    /// App-Logik darf NICHT davon abhängen (unzuverlässige Ausführung).
    static func registerBackgroundTask() {
        BGTaskScheduler.shared.register(
            forTaskWithIdentifier: bgTaskIdentifier,
            using: nil
        ) { task in
            guard let refreshTask = task as? BGAppRefreshTask else { return }
            Task { @MainActor in
                // Best-Effort: Container und Repo werden vom App-Kontext geholt,
                // falls nicht verfügbar: kein Crash, nur kein Update
                refreshTask.setTaskCompleted(success: true)
            }
        }
    }

    /// Plant den nächsten BGAppRefreshTask (nächste volle Stunde).
    static func scheduleBackgroundRefresh() {
        let request = BGAppRefreshTaskRequest(identifier: bgTaskIdentifier)
        let nextHour = Calendar.current.nextDate(
            after: Date(),
            matching: DateComponents(minute: 0),
            matchingPolicy: .nextTime
        ) ?? Date().addingTimeInterval(3600)
        request.earliestBeginDate = nextHour
        try? BGTaskScheduler.shared.submit(request)
    }
    #endif
}
```

## Expected Behavior

- **Input:** `reconcile(reason:container:eventKitRepo:)` wird von `FocusBloxApp` bei `scenePhase == .active` und `scenePhase == .background` aufgerufen
- **Output:** Alle pending Notifications werden gecleared und neu berechnet. Nach Reconciliation sind maximal 64 Requests pending — Prio-1-Slots (Timer) immer zuerst belegt, dann Prio-2 (Tasks) nur bei Profil `balanced` oder `active`
- **Side effects:**
  - `removeAllPendingNotificationRequests()` löscht vorherige Queue vollständig
  - BGAppRefreshTask wird bei App-Start registriert und für nächste volle Stunde geplant
  - `rescheduleDueDateNotifications()` in `FocusBloxApp` existiert nach Änderung nicht mehr
  - `NotificationService` bleibt unverändert — kein bestehender Test bricht

## Identifier-Konvention

Alle neuen Notifications folgen diesem Schema (rückwärtskompatibel mit bestehenden IDs aus `NotificationService`, da Phase A `build*Request`-Methoden direkt nutzt):

| Kategorie | ID-Prefix | Quelle |
|-----------|-----------|--------|
| Focus Block Timer (Start) | `focus-block-start-{blockID}` | `NotificationService.buildFocusBlockNotificationRequest` |
| Focus Block Timer (End) | `focus-block-end-{blockID}` | `NotificationService.buildFocusBlockEndNotificationRequest` |
| Task Due Date Morning | `due-date-morning-{taskID}` | `NotificationService.buildDueDateMorningRequest` |
| Task Due Date Advance | `due-date-advance-{taskID}` | `NotificationService.buildDueDateAdvanceRequest` |
| Review / Morning | `focusblox.review.{date}` / `focusblox.morning.{date}` | Phase C |
| Nudges | `focusblox.nudge.{type}.{date}` | Phase C |

## Known Limitations

- Phase A implementiert `buildReviewRequests()` und `buildNudgeRequests()` als leere Platzhalter — Prio-3 und Prio-4 Slots werden erst in Phase C befüllt
- `BGAppRefreshTask` in `reconcile()` erhält keinen vollständigen Container/Repo-Zugriff ohne App-Kontext — die Background-Reconciliation ist in Phase A ein No-Op (Best-Effort-Registrierung ist trotzdem vorhanden)
- `NotificationActionDelegate` bleibt in Phase A unberührt — er rescheduled noch einzeln. Volle Anbindung in Phase B/C
- macOS: kein `BGAppRefreshTask`, kein Badge-Update — `#if !os(macOS)` Guards schützen alle iOS-only APIs
- Bestehende `task-timer-*` Identifiers (aus `scheduleTaskOverdueNotification`) werden durch `removeAllPendingNotificationRequests()` bei jeder Reconciliation gecleared — diese Notifications müssen danach von der jeweiligen View neu geplant werden, bis Phase B die direkte View-Integration ablöst
