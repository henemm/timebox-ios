# RW_0.1b Phase B — Smart Notification Engine FocusBlock Migration

## Approval

- [ ] Approved

## Purpose

Ersetzt alle direkten `NotificationService.schedule/cancel`-Aufrufe in FocusBlock-bezogenen Views durch `SmartNotificationEngine.reconcile()`. Integriert ausserdem Task-Overdue-Notifications als dedizierte Engine-Methoden. Nach Phase B gibt es keinen direkten Notification-Zugriff mehr in Views — die Engine ist der einzige Einstiegspunkt.

## Implementation Details

### 1. `Sources/Services/SmartNotificationEngine.swift` (~+30 LoC, MODIFY)

Drei neue Ergaenzungen nach dem bestehenden `reconcile(reason:container:eventKitRepo:)`:

**1a. ModelContext-Overload fuer Views:**

```swift
// MARK: - Reconcile (ModelContext Overload — fuer Views ohne ModelContainer-Zugriff)

static func reconcile(
    reason: ReconciliationReason,
    context: ModelContext,
    eventKitRepo: any EventKitRepositoryProtocol
) async {
    let start = Date()
    let center = UNUserNotificationCenter.current()

    center.removeAllPendingNotificationRequests()

    let requests = await buildAllRequests(
        profile: AppSettings.shared.notificationProfile,
        context: context,
        eventKitRepo: eventKitRepo
    )

    let capped = Array(requests.prefix(64))
    for request in capped {
        try? await center.add(request)
    }

    let elapsed = Date().timeIntervalSince(start) * 1000
    print("SmartNotificationEngine: reconcile(\(reason)) — \(capped.count) notifications, \(String(format: "%.1f", elapsed))ms")
}
```

**1b. `buildAllRequests`-Overload mit ModelContext:**

```swift
static func buildAllRequests(
    profile: NotificationProfile,
    context: ModelContext,
    eventKitRepo: any EventKitRepositoryProtocol
) async -> [UNNotificationRequest] {
    var requests: [UNNotificationRequest] = []

    requests += buildTimerRequests(eventKitRepo: eventKitRepo)

    if profile == .balanced || profile == .active {
        requests += buildTaskRequests(context: context)
    }

    if profile == .balanced || profile == .active {
        requests += buildReviewRequests()
    }

    if profile == .active {
        requests += buildNudgeRequests()
    }

    return Array(requests.prefix(64))
}
```

**1c. `buildTaskRequests`-Overload mit ModelContext (kein `ModelContext(container)` Umweg):**

```swift
private static func buildTaskRequests(
    context: ModelContext
) -> [UNNotificationRequest] {
    let settings = AppSettings.shared
    guard settings.dueDateMorningReminderEnabled || settings.dueDateAdvanceReminderEnabled else {
        return []
    }

    let descriptor = FetchDescriptor<LocalTask>(
        predicate: #Predicate<LocalTask> { !$0.isCompleted && !$0.isTemplate && $0.dueDate != nil },
        sortBy: [SortDescriptor(\.dueDate)]
    )
    guard let tasks = try? context.fetch(descriptor) else { return [] }

    let now = Date()
    var requests: [UNNotificationRequest] = []

    for task in tasks {
        guard let dueDate = task.dueDate, dueDate > now else { continue }
        if settings.dueDateMorningReminderEnabled,
           let req = NotificationService.buildDueDateMorningRequest(
               taskID: task.id,
               title: task.title,
               dueDate: dueDate,
               morningHour: settings.dueDateMorningReminderHour,
               morningMinute: settings.dueDateMorningReminderMinute,
               now: now
           ) {
            requests.append(req)
        }
        if settings.dueDateAdvanceReminderEnabled,
           let req = NotificationService.buildDueDateAdvanceRequest(
               taskID: task.id,
               title: task.title,
               dueDate: dueDate,
               advanceMinutes: settings.dueDateAdvanceReminderMinutes,
               now: now
           ) {
            requests.append(req)
        }
        if requests.count >= budgetTasks { break }
    }

    return Array(requests.prefix(budgetTasks))
}
```

**1d. Task-Overdue-Methoden (Laufzeit-State, nicht Teil von reconcile):**

```swift
// MARK: - Task Overdue (Laufzeit-State — nicht Teil von reconcile)

/// Plant eine Overdue-Notification fuer den laufenden Task.
/// Wird von FocusLiveView.trackTaskStart aufgerufen.
/// Nicht Teil von reconcile() — ist Laufzeit-State, kein Gesamt-State.
static func scheduleTaskOverdue(
    taskID: String,
    taskTitle: String,
    durationMinutes: Int
) {
    NotificationService.scheduleTaskOverdueNotification(
        taskID: taskID,
        taskTitle: taskTitle,
        durationMinutes: durationMinutes
    )
}

/// Bricht eine laufende Overdue-Notification ab.
/// Wird bei Task-Wechsel oder Task-Abschluss aufgerufen.
static func cancelTaskOverdue(taskID: String) {
    NotificationService.cancelTaskNotification(taskID: taskID)
}
```

## Expected Behavior

- **Input:** Jede Block-Mutation (create/update/delete) oder Task-Statusaenderung in einer der betroffenen Views
- **Output:** `reconcile()` berechnet alle 64 Slots neu — keine inkonsistenten Einzelnotifications mehr
- **Side effects:**
  - `rescheduleEndNotification(for:)` in `FocusLiveView` existiert nach Migration nicht mehr
  - `rescheduleDueDateNotifications()` in `FocusBloxMacApp` existiert nach Migration nicht mehr
  - 16 direkte `NotificationService`-Aufrufe in Views werden entfernt
  - `NotificationService.schedule/cancel`-Methoden bleiben weiterhin erhalten (Phase C entscheidet ueber vollstaendige Deprecation)
  - Task-Overdue-Notifications werden weiterhin individuell verwaltet (kein Full-Reconcile bei Task-Start)

## Known Limitations

- Jede Block-Aktion loest Full-Reconcile aus (alle 64 Slots neu). Phase A Tests zeigen <50ms — akzeptabel.
- `NotificationActionDelegate` hat eigene schedule-Calls — wird nicht in Phase B angefasst (Phase C).
- `scheduleTaskOverdue` / `cancelTaskOverdue` sind Thin Wrappers ohne eigene Logik — dienen als Abstraktions-Layer fuer Phase C (wenn Overdue-Logik in Engine gezogen wird).
- macOS nutzt weiterhin `reconcile(reason:container:eventKitRepo:)` — der neue `ModelContext`-Overload ist fuer iOS-Views.
