# Story 3.1: Task direkt auf Kalender droppen

**Epic:** 3 — Fokussierte Ausführung
**Priorität:** Hoch | **Abhängigkeiten:** Keine

## Ziel

Task per Drag&Drop auf Kalender-Timeline einplanen — ohne Focus Block. Long-Press bietet Wahl zwischen "Task einplanen" und "Focus Block erstellen".

## Akzeptanzkriterien

- [ ] Drop auf Timeline: Task wird zeitlich fixiert (kein Focus Block)
- [ ] Geplante Zeit = Drop-Position, Dauer = `estimatedDuration` oder Default 30 Min.
- [ ] Long-Press auf Drop-Zone: Kontext-Menü "Task einplanen" vs. "Focus Block erstellen"
- [ ] Geplante Tasks in Timeline als eigener visueller Typ (unterscheidbar von Focus Blocks und Events)
- [ ] Tasks per Drag verschiebbar, per Swipe/Tap enttimbar
- [ ] Kollisionserkennung mit Events/Focus Blocks (visueller Hinweis, kein Blocking)
- [ ] Geplante Tasks triggern Notification-Reconciliation (→ Story 0.1)

## Technische Vorgaben

### Model-Änderung: `LocalTask`
```swift
var scheduledDate: Date?      // nil = ungeplant
var scheduledDuration: Int?   // Override für estimatedDuration in Minuten
```

### Neue Dateien
| Datei | Beschreibung |
|---|---|
| `ScheduledTaskBlock.swift` | Timeline-Darstellung eines geplanten Tasks |

### Betroffene Dateien
| Datei | Änderung |
|---|---|
| `LocalTask.swift` | + `scheduledDate`, + `scheduledDuration` |
| `PlanningView.swift` | Drop-Handler: Task-Drop ohne Focus Block |
| `TimelineView.swift` | + ScheduledTaskBlock Rendering |
| `TimelineItem.swift` | + Case für scheduled Tasks |
| `GapFinder.swift` | Scheduled Tasks als belegte Slots |
| `BlockPlanningView.swift` | Long-Press Kontext-Menü |
| `MacPlanningView.swift` | Gleiches Drop-Verhalten |
| `MacTimelineView.swift` | + ScheduledTaskBlock |

### Constraints
- Task kann NICHT gleichzeitig `scheduledDate` haben UND in einem `FocusBlock` sein
- CloudKit-Sync für neue Felder
- SmartNotificationEngine: Reconciliation bei Schedule-Änderung
