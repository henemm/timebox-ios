# Story 3.2: Focus Sprint ("Los"-Button)

**Epic:** 3 — Fokussierte Ausführung
**Priorität:** Hoch | **Abhängigkeiten:** Story 3.1

## Ziel

Direkter Start eines Focus Blocks aus Backlog/Next-Up/Scheduled heraus — ein Tap.

## Akzeptanzkriterien

- [ ] "Los"-Button an jedem Task (Backlog, Next-Up, Scheduled, Tagesansicht)
- [ ] Tap erstellt Focus Block: Start = jetzt, Dauer = `estimatedDuration` oder Default 60 Min.
- [ ] Optional: Dauer vor Start anpassen (Inline-Picker, kein Sheet)
- [ ] App wechselt in FocusLiveView, alle anderen Tasks ausgeblendet
- [ ] Live Activity + Dynamic Island aktiviert
- [ ] Bei Abschluss: normaler Sprint-Review-Flow
- [ ] Warnung wenn bereits ein Focus Block aktiv ist
- [ ] Wenn Task `scheduledDate` hat: wird beim Start gelöscht

## Technische Vorgaben

### Betroffene Dateien
| Datei | Änderung |
|---|---|
| `BacklogRow.swift` | + "Los"-Button |
| `NextUpSection.swift` | + "Los"-Button pro Task |
| `MorningCoachingSection.swift` | + "Los"-Button an Vorschlägen |
| `FocusBlockActionService.swift` | + `startImmediate(task:duration:)` |
| `MainTabView.swift` | Auto-Switch zu Focus-Tab |
| `MacBacklogRow.swift` | + "Los" im Context Menu |

### Constraints
- `startImmediate` nutzt denselben Code-Pfad wie manuelle Block-Erstellung (DRY)
- Kein zweiter paralleler Focus Block erlaubt
- SmartNotificationEngine: Reconciliation bei Block-Start
