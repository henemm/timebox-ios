# Story 3.3: Follow-up Logic

**Epic:** 3 — Fokussierte Ausführung
**Priorität:** Mittel | **Abhängigkeiten:** Story 3.2

## Ziel

Beim Abbruch eines Focus Blocks: Stand festhalten, automatisch Anschlussaufgabe erstellen.

## Akzeptanzkriterien

- [ ] Bei Abbruch: "Wie weit bist du?" (Freitext, optional)
- [ ] Automatische Anschlussaufgabe: "Weiter: [Original-Titel]"
- [ ] Erbt: Kategorie, Importance, Tags
- [ ] Restdauer = Original minus abgelaufen (Minimum 15 Min.)
- [ ] Landet im Backlog mit `lifecycleStatus = .active`
- [ ] Follow-up-Kette: Verweist immer auf ORIGINAL-Task (flach, kein Baum)

## Technische Vorgaben

### Model-Änderung: `LocalTask`
```swift
var parentTaskID: String?    // Verweis auf Original-Task
var progressNote: String?    // Fortschritts-Notiz bei Abbruch
```

### Betroffene Dateien
| Datei | Änderung |
|---|---|
| `LocalTask.swift` | + `parentTaskID`, + `progressNote` |
| `FocusBlockActionService.swift` | + `abortWithFollowUp(task:note:elapsedMinutes:)` |
| `SprintReviewSheet.swift` | + Abbruch-Flow mit Freitext |
