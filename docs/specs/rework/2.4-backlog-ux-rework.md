# Story 2.4: Backlog UX Rework

**Epic:** 2 — Adaptive Tagesplanung
**Priorität:** Niedrig | **Abhängigkeiten:** Story 1.3

## Ziel

Backlog-Ansicht mit Parkdeck-Metapher: Aktive Tasks prominent, selten relevante Tasks visuell zurückgestuft.

## Akzeptanzkriterien

- [ ] Default-Ansicht zeigt nur `doNow` und `planSoon` ("Aktive Tasks")
- [ ] `eventually`/`someday` hinter ausklappbarem "Parkdeck"-Bereich (collapsed by default)
- [ ] Parkdeck zeigt Badge mit Anzahl, nicht die Tasks selbst
- [ ] Swipe/Drag zum Parken oder Aktivieren
- [ ] Suchfunktion durchsucht immer ALLE Tasks
- [ ] Bestehende 5 View-Modi bleiben als Filter verfügbar

## Technische Vorgaben

### Betroffene Dateien
| Datei | Änderung |
|---|---|
| `BacklogView.swift` | Aktiv-Sektion + Parkdeck-Sektion |
| `BacklogRow.swift` | + Swipe-Action "Parken" / "Aktivieren" |
| `MacBacklogRow.swift` | + Context Menu Einträge |

### Constraints
- Parkdeck-Bereich: Lazy Loading (nicht rendern wenn collapsed)
- Keine Änderung an `TaskPriorityScoringService` Scoring-Logik
