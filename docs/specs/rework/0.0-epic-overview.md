## Architektur-Referenz

- **Pattern:** Service-Oriented (keine ViewModels, Services sind First-Class)
- **Daten:** SwiftData + CloudKit Sync (Private DB: `iCloud.com.henning.focusblox`)
- **Settings:** AppSettings (UserDefaults) + SyncedSettings (iCloud KV)
- **Navigation:** iOS = TabView | macOS = Sidebar + Content
- **AI:** Apple Intelligence via Foundation Models (~3B Parameter, on-device)
- **Target:** iOS 26.2 / macOS 26.2 / watchOS 26.2

## Zentrale Design-Entscheidungen

| # | Entscheidung | Beschluss |
|---|---|---|
| D1 | Parkdeck vs. Backlog | Kein separates Parkdeck. Backlog bleibt, Priority-Scoring (doNow/planSoon/eventually/someday) bleibt Basis. UX bekommt Parkdeck-Metapher (visuelles Verstecken von eventually/someday). |
| D2 | Task zeitlich einplanen | Task kann per Drop auf Kalender-Timeline eingeplant werden OHNE Focus Block. Long-Press: "Task einplanen" vs. "Focus Block erstellen". |
| D3 | Evening Reset | Soft Reset: Unerledigte Next-Up-Tasks → zurück ins Backlog, rescheduleCount++, Prio steigt. Next-Up morgens leer. |
| D4 | Focus Blocks | Bleiben als Power-Feature (Timer, Gong, Live Activity). Nicht mehr Pflicht für zeitliche Planung. |
| D5 | Tagesansicht | Neuer Screen mit zwei Modi: Morgens = Coaching + Vorschläge, Abends = Reflexion + Success Story. Ersetzt NICHT Backlog/Kalender — ist Klammer um den Tag. |
| D6 | Manuelle vs. geführte Planung | Beides vollwertig. Nutzer kann KI-Vorschläge nutzen ODER selbst im Backlog planen und Tasks in Kalender ziehen. |
| D7 | Apple Intelligence Scope | On-Device-Modell (~3B) ist gut für Extraktion, Klassifikation, kurze Textgenerierung. NICHT für Reasoning oder Verhaltensvorhersage. Intelligente Vorschläge kommen aus deterministischen Services, LLM ist nur die "Stimme". |
| D8 | Notifications | Harte iOS-Grenze: max 64 pending Notifications. Kein Pre-Delivery-Check möglich. Strategie: Reconcile bei jedem App-Event. Drei Nutzer-Profile (Leise/Ausgeglichen/Aktiv). |
