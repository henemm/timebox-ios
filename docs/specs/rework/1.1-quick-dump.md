# Story 1.1: Quick Dump

**Epic:** 1 — Reibungslose Erfassung & Smarte Veredelung
**Priorität:** Hoch | **Abhängigkeiten:** Keine

## Ziel

Nutzer erfasst eine Aufgabe mit minimalem Aufwand (Text oder Sprache), ohne Workflow-Unterbrechung. Erfassung in < 3 Sekunden.

## Kontext

`QuickCaptureView` (iOS), `QuickCapturePanel` (macOS, Spotlight-Style), `VoiceInputSheet` (watchOS) existieren. Diese Story vereinfacht den Flow: Nur Textfeld, kein Formular. Task landet im Refiner (Story 1.3), nicht im Backlog.

## Akzeptanzkriterien

- [ ] Erfassung benötigt nur ein Textfeld (kein Pflichtfeld außer Titel)
- [ ] Spracheingabe als primäre Methode verfügbar (iOS: Diktat-Button, watchOS: VoiceInputSheet)
- [ ] Task wird mit `lifecycleStatus = .raw` gespeichert
- [ ] Task erscheint NICHT im Backlog, sondern im Refiner (→ 1.3)
- [ ] Erfassung ist in < 3 Sekunden abgeschlossen (inline oder Overlay, kein Sheet mit Formular)
- [ ] Siri: `CreateTaskIntent` akzeptiert Freitext, setzt `lifecycleStatus = .raw`
- [ ] Share Extension setzt ebenfalls `lifecycleStatus = .raw`
- [ ] Nach Erfassung: Badge am Refiner-Tab/Indikator zeigt Anzahl unveredelter Tasks

## Technische Vorgaben

### Model-Änderung: `LocalTask`
```swift
enum TaskLifecycleStatus: String, Codable {
    case raw       // Gerade erfasst, nicht veredelt
    case refined   // Durch Refiner bestätigt
    case active    // Im Backlog / Next-Up
}
```
- Neues Feld: `lifecycleStatus: TaskLifecycleStatus` (Default: `.raw`)
- Migration: Alle bestehenden Tasks bekommen `.active`

### Betroffene Dateien
| Datei | Änderung |
|---|---|
| `LocalTask.swift` | + `lifecycleStatus`, + `TaskLifecycleStatus` Enum |
| `QuickCaptureView.swift` | Vereinfachen: nur Textfeld + Mikrofon |
| `QuickCapturePanel.swift` | Gleiches Verhalten wie iOS |
| `CreateTaskIntent.swift` | Freitext-Modus, setzt `.raw` |
| `ShareViewController.swift` (iOS + macOS) | Setzt `.raw` |
| `BacklogView.swift` | Tasks mit `.raw` ausfiltern |

### Constraints
- CloudKit-Sync muss `lifecycleStatus` synchronisieren
- watchOS: Bestehende `VoiceInputSheet` nutzen, Status `.raw` setzen
- Keine neue Dependency
