dasselbe Muster und ist laut git history bereits gruen (stand vor FEATURE_005).
KEIN BUG.

BEFUND 2: FUNKTIONAL — Import-Button .disabled() fehlt
---------------------------------------------------------
Vergleich Normal-Backlog (ContentView.swift Zeile 572):
  .disabled(isSyncing)   ← VORHANDEN

Coach-Toolbar (MacCoachBacklogView.swift Zeile 94-102):
  if remindersSyncEnabled {
      Button {
          Task { await onImport?() }
      } label: {
          Image(systemName: "square.and.arrow.down")
      }
      .accessibilityIdentifier("coachImportRemindersButton")
      .help("Erinnerungen importieren")
      // KEIN .disabled(isSyncing) !!!
  }

Der normale Backlog-Import-Button ist disabled waehrend ein Import laeuft.
Der Coach-Import-Button nicht. User kann Import mehrfach gleichzeitig starten.
Das ist eine funktionale Regression gegenueber dem normalen Backlog.

BEFUND 3: FUNKTIONAL — isSyncing-State fehlt in MacCoachBacklogView
--------------------------------------------------------------------
Der normale Backlog hat ZWEI Sync-Indikatoren:
  1. cloudKitMonitor.isSyncing  (CloudKit background sync)
  2. isSyncing                  (lokaler Reminders-Import-Zustand, @State in ContentView)

ContentView.swift Zeile 537:
  if cloudKitMonitor.isSyncing || isSyncing {
      ProgressView()...

MacCoachBacklogView.swift Zeile 351:
  if cloudKitMonitor.isSyncing {    ← NUR CloudKit, kein lokaler isSyncing!
      ProgressView()...

MacCoachBacklogView bekommt keinen isSyncing-State von ContentView.
Die View-Signatur hat kein isSyncing-Parameter.
Konsequenz: Waehrend Reminders importiert werden, zeigt Coach-Toolbar
keinen Ladeindikator. Normaler Backlog schon.
